# senior-backend-prep
Structured roadmap of projects and study to transition into Senior Backend Engineering, with a focus on networking, distributed systems, and real-time backends.
This repo serves as both a **portfolio** and a **learning log**, showcasing practical work across networking, APIs, microservices, distributed systems, and real-time backends.

---

## Purpose
- Demonstrate a deliberate, step-by-step path into backend engineering.
- Showcase projects that mirror real industry scenarios (scalable APIs, real-time services, cloud deployments).
- Build a portfolio that highlights not just code, but **design thinking, documentation, and trade-off decisions**.

---

## Structure
This repository is organized by topic, following a 6-month transition plan:

1. **Networking Basics**
   - TCP/UDP echo servers (Python & C#)
   - Asynchronous socket chat server
   - Notes on TCP vs UDP tradeoffs
2. **Backend Foundations**
   - REST API with database + JWT authentication
   - CRUD operations, error handling
   - Dockerized local deployment
3. **Distributed Systems**
   - Microservices split from API
   - Message queues (RabbitMQ/Kafka/SQS)
   - Redis caching layer
4. **Cloud & DevOps**
   - CI/CD pipelines (GitHub Actions)
   - Deployment to AWS / Azure
   - Monitoring + structured logging
5. **Real-Time Systems**
   - WebSocket multiplayer lobby
   - Matchmaking logic + state management
   - Load testing (50â€“100 simulated players)
6. **Advanced Topics**
   - CAP theorem + consistency models
   - Event sourcing & CQRS
   - Reliability patterns (retry, backoff, circuit breaker)

---

## Checkpoints
Each phase ends with a checkpoint:
- Clear explanation of core concepts
- Working code with documentation
- Reflection on trade-offs and lessons learned
